# Формат ответа ?jobs

Фронтенд опрашивает `index.php?jobs` каждые 1.5-12 секунд и получает JSON со всем, что нужно для перерисовки таблиц. Эндпоинт ничего не рендерит - только отдаёт данные.

## Запрос

```
GET /index.php?jobs
```

Ответ - `application/json`.

## Форма ответа

```json
{
  "jobs": [],
  "queue": [
    {
      "pid": "queueidXXXX",
      "url": "https://example.com/video",
      "dl_format": "Топ",
      "type": "video"
    }
  ],
  "finished": [],
  "videos": [],
  "music": [],
  "logURL": "",
  "proxy": {}
}
```

| Поле | Тип | Что содержит |
|---|---|---|
| `jobs` | array | Активные (ещё идущие) загрузки - результат `Downloader::get_current_background_jobs()`. |
| `queue` | array | Загрузки, ожидающие своей очереди. Пустой массив, если `disableQueue` включён в конфиге. Каждый элемент - `pid`, `url`, `dl_format` (человекочитаемое название формата, например "Топ", "4K", "Булшит"), `type` (`video` или `audio`). |
| `finished` | array | Завершённые загрузки (успешные и с ошибкой) - результат `Downloader::get_finished_background_jobs()`, с кешированием разбора логов. |
| `videos` | array | Список файлов из папки видео (`FileHandler::listVideos()`), с полями строки файла (см. ниже). |
| `music` | array | Список файлов из папки музыки (`FileHandler::listMusics()`), той же формы. |
| `logURL` | string | Относительный путь к логам, если `logURL` в конфиге не пустой. Иначе пустая строка. |
| `proxy` | object | Статус живости SOCKS5-прокси - результат `ProxyStatus::payload()` (окна 1/5/15 минут). |

## Поля одной строки файла (videos / music)

Формируются функцией `generateFileRow()` в `index.php`. Ключевые поля:

| Поле | Что содержит |
|---|---|
| `downloadurl` | Относительный путь к файлу - используется и как ссылка на скачивание, и как ключ строки для анимации появления новых файлов, и для QR-кода. |
| `kind` | `'video'` или `'audio'` - используется встроенным плеером, чтобы показать нужный тег (`<video>`/`<audio>`). |

## Дополнительный режим ?cron

Если к запросу добавить `&cron`, эндпоинт всё так же продвигает очередь загрузок (`process_queue()`), но не отдаёт JSON и не завершает вывод с `Content-Type: application/json` - используется, чтобы гарантированно двигать очередь по расписанию, даже если никто не смотрит на страницу.

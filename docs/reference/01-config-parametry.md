# Параметры конфигурации

Все настройки сайта лежат в одном файле - `app/var/www/YT/config/config.php`. Файл возвращает обычный PHP-массив с комментариями к каждому параметру.

| Параметр | По умолчанию | Что значит |
|---|---|---|
| `siteName` | `'Качалка v.1.0b'` | Название, которое видно в заголовке страницы и в `<title>`. |
| `disableExtraction` | `false` | Если `true` - убирает возможность конвертации в аудио. |
| `max_dl` | `1` | Сколько загрузок может идти одновременно. `-1` убирает лимит совсем. |
| `outputFolder` | `/var/www/YT/download` | Абсолютный путь, куда сохраняются скачанные файлы. |
| `downloadPath` | `'download'` | Относительный веб-путь к папке загрузок. Пустое значение - ссылки на скачивание не показываются. |
| `redirectAfterSubmit` | `'downloads'` | На какую вкладку перекинуть пользователя после отправки ссылки на загрузку. |
| `logPath` | `/var/www/YT/tmp` | Папка, куда yt-dlp и сам сайт пишут служебные файлы: прогресс, очередь, PID-файлы. |
| `logURL` | `''` | Если эта папка доступна через веб - относительный путь до неё, чтобы в интерфейсе появилась ссылка на лог загрузки. Пустое значение - логи не публикуются. |
| `youtubedlExe` | `/.yt_env/bin/yt-dlp` | Путь к исполняемому файлу yt-dlp. |
| `maxVideoRes` | `1080` | Потолок качества видео при обычном нажатии кнопки "Скачать". Долгое нажатие позволяет выбрать качество явно, поверх этого потолка. |
| `socks5` | из `.env` | Адрес SOCKS5-прокси. Читается через `getenv('SOCKS5_URL')`. |
| `proxyStatus` | `true` | Показывать ли индикатор живости прокси в футере. Работает только если `socks5` не пустой. |
| `keepPartialFiles` | `false` | Оставлять ли недокачанные `.part`-файлы после нажатия "Остановить всё". |
| `disableQueue` | `false` | Если `true` - при превышении лимита `max_dl` загрузка не запустится, вместо того чтобы встать в очередь. |
| `allowFileDelete` | `true` | Разрешить ли пользователям удалять файлы из интерфейса. |
| `showFileLifetime` | `true` | Показывать ли таймер "сколько осталось до автоудаления" у каждого файла. Имеет смысл только если настроен хостовый cron очистки. |
| `retentionMinutes` | `120` | Окно жизни файла в минутах - шкала для таймера в интерфейсе. Должно совпадать с окном очистки в хостовом cron (`2hourcleanup.sh`, `find -mmin +N`). |
| `debug` | `false` | Включает вывод всех PHP-ошибок на экран. Только для разработки. |

## Прокси и .env

`.env.example` показывает, какую переменную нужно задать:

```
SOCKS5_URL=socks5://user:pass@host:port
```

Реальный файл `.env` не должен попадать в git - он подключается через `env_file` в `docker-compose.yml`. Как настроить прокси по шагам - в [Настроить SOCKS5-прокси](../how-to/03-socks5-proxy.md).

## Прямой доступ и доступ через прокси

В классе `Downloader` (`class/Downloader.php`) есть список доменов `DIRECT_ACCESS_DOMAINS` - российские и не блокирующие сайты (vk.com, rutube.ru, ok.ru и другие). Ссылки на эти домены и их поддомены скачиваются напрямую, без прокси. Все остальные ссылки, включая YouTube, идут через прокси, если он настроен.

## Фильтр 18+ в yt-dlp.conf

Файл `app/etc/yt-dlp.conf` - общий конфиг, который yt-dlp подхватывает при каждом запуске. В нём есть `--match-filter` - регулярное выражение со списком доменов для взрослых. Если ссылка ведёт на сайт из этого списка, yt-dlp откажется её скачивать ещё до старта загрузки.

## nginx

Конфиг лежит в `app/etc/nginx/sites-available/default`:

- Запросы к `.php`-файлам уходят в php-fpm через unix-сокет.
- Папки `tmp`, `class`, `config` и `.yt_env` полностью закрыты от прямого доступа через веб.
- Файлы `.php` в папках `download/` и `favicons/` не исполняются, отдаются только как статика.
- Картинки и шрифты кешируются на неделю, CSS и JS - на сутки, иконки сайтов - на год.
- Таймаут ответа от PHP ограничен 30 секундами.

Для проверки живости контейнера есть лёгкий эндпоинт `health.php` - отдаёт только `200 OK`, не запуская фронт-контроллер. Именно его дёргает Docker `HEALTHCHECK`.

# Настроить сайт под себя

Все настройки сайта лежат в одном файле - `app/var/www/YT/config/config.php`. Полный список параметров с описанием - в [Параметрах конфигурации](../reference/01-config-parametry.md). Здесь - готовые рецепты для частых задач.

## Поменять название сайта

```php
'siteName' => 'Моя качалка',
```

## Ограничить качество видео по умолчанию

```php
'maxVideoRes' => 1080,
```

Это ограничивает только обычную кнопку "Скачать". Долгое удержание кнопки всё равно позволит выбрать явное качество (4K, 2K, 1080p), даже выше этого потолка - так задумано.

## Разрешить или запретить несколько загрузок одновременно

```php
'max_dl' => 1,
```

`1` - одна загрузка за раз, остальные ждут очереди. `-1` снимает лимит совсем - не рекомендуется, особенно если качаешь с YouTube (см. [что влияет на скорость бана IP](../explanation/06-youtube-i-perevod-kak-eto-rabotaet.md#что-влияет-на-скорость-бана-ip)).

## Отключить конвертацию в аудио

Если ffmpeg недоступен или не хочется тратить на конвертацию ресурсы сервера:

```php
'disableExtraction' => true,
```

## Поменять папку для скачанных файлов

```php
'outputFolder' => '/var/www/YT/download',
'downloadPath' => 'download',
```

`outputFolder` - абсолютный путь на диске, `downloadPath` - относительный веб-путь, по которому файл можно скачать прямо по ссылке из браузера. Если оставить `downloadPath` пустым, ссылки на скачивание вообще не показываются - подходит, если файлы забираются каким-то другим способом (например, по сети).

## Поменять время жизни файлов

```php
'retentionMinutes' => 120,
```

Это только цифра для таймера в интерфейсе - реальное удаление делает хостовый cron (см. [Установка через Docker](01-ustanovka-docker.md#настрой-автоочистку-и-автообновление)). Если меняешь это значение, поменяй тот же интервал (`find -mmin +N`) и в скрипте `2hourcleanup.sh`, иначе таймер на сайте разойдётся с реальным удалением.

## Запретить пользователям удалять файлы

```php
'allowFileDelete' => false,
```

## Включить debug-режим (только для разработки)

```php
'debug' => true,
```

Показывает все ошибки PHP прямо на странице. Ни в коем случае не включай на сервере, доступном из интернета - это раскрывает внутренние пути и детали реализации.

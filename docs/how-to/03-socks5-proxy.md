# Настроить SOCKS5-прокси

Прокси нужен, если ты хочешь скачивать с сайтов, заблокированных по региону (например, YouTube с российского IP), или наоборот - обходить блокировку российских сайтов из-за рубежа.

## Шаг 1. Пропиши адрес прокси в .env

Если файла `.env` ещё нет, создай его из примера:

```bash
cp .env.example .env
```

Открой `.env` и укажи адрес:

```
SOCKS5_URL=socks5://user:pass@host:port
```

Если строку оставить пустой, прокси не используется вообще - все загрузки идут напрямую.

## Шаг 2. Перезапусти контейнер

```bash
docker-compose up -d
```

или, если контейнер запущен напрямую через `docker run`, останови его и запусти заново - переменная окружения читается при старте.

## Шаг 3. Проверь, что прокси подхватился

В подвале страницы (панель "Жмак") появится индикатор из трёх точек - окна замеров живости прокси за 1, 5 и 15 минут. Зелёный - прокси работает, жёлтый - есть отдельные сбои, красный - недоступен, серый - данные ещё не собрались (подожди минуту-две после запуска).

Если индикатора нет вообще - проверь, что `SOCKS5_URL` не пустая строка и параметр `proxyStatus` в `config.php` не выключен (см. [Параметры конфигурации](../reference/01-config-parametry.md)).

## Как это работает под капотом

Не все ссылки идут через прокси - какие именно и почему, разобрано в [Backend](../explanation/02-backend.md#список-доменов-прямого-доступа). Что происходит, если загрузка с "прямого" домена вдруг падает с сетевой ошибкой - в [Автоматическом ретрее через прокси при блокировке](../explanation/04-ochered-i-fishki.md#автоматический-ретрей-через-прокси-при-блокировке).

Пароль от прокси нигде не хранится в коде проекта и не попадает в git - он живёт только в `.env` и подставляется в команду yt-dlp во время выполнения.

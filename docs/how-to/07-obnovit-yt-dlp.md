# Обновить yt-dlp

Самая частая причина "перестало качать" - устаревший yt-dlp: сайты постоянно меняются, а YouTube особенно активно борется с загрузчиками. Свежая версия обычно чинит проблему.

## Автоматическое обновление (уже настроено)

Если ты настроил хостовый cron по инструкции [Установка через Docker](01-ustanovka-docker.md#настрой-автоочистку-и-автообновление), yt-dlp обновляется сам каждый день в 03:00. Версия, которую видно в подвале страницы, обновится после следующей перезагрузки страницы.

## Обновить прямо сейчас, вручную

Если не хочется ждать ночного обновления:

```bash
docker exec -u root yt /etc/Scripts/update-ytdlp.sh
```

Команда обязательно с `-u root` - скрипт пишет в Python-окружение `/.yt_env`, владелец которого root.

## Обновить пересборкой образа

Более радикальный способ - собрать образ заново, что заодно подтянет свежие версии всех остальных инструментов (bgutil, vot-cli):

```bash
docker build --no-cache -t yt . && docker-compose up -d
```

## Если после обновления пропали имена файлов или появились странные символы

При обновлении yt-dlp накатывается патч, который делает очистку имён файлов мягче (двоеточие превращается в дефис, а не в подчёркивание). Если `pip install` перезаписывает исходный файл yt-dlp, патч слетает. Скрипт `update-ytdlp.sh` переналагает патч автоматически на каждое обновление, но если что-то пошло не так, переналожить его вручную можно так:

```bash
docker exec yt patch_replace_insane.sh container
```

## Если обновление упало с ошибкой

Скрипт использует `set -e` - при сбое (например, upstream переписал код и патч больше не накладывается) он завершится с ошибкой в логе, но контейнер продолжит работать на текущей версии yt-dlp. Это не авария - просто версия останется прежней до следующей успешной попытки. Проверить лог обновления:

```bash
tail -f /var/log/yt/ytdlp-update.log
```

Если патч действительно перестал накладываться - потребуется вручную перегенерировать `app/patches/replace_insane.patch` под новую версию yt-dlp.

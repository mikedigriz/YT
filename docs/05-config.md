# Конфигурация

Все настройки сайта лежат в одном файле - `app/var/www/YT/config/config.php`. Файл возвращает обычный PHP-массив с комментариями к каждому параметру.

## Параметры config.php

| Параметр | По умолчанию | Что значит |
|---|---|---|
| `siteName` | `'Качалка v.1.0b'` | Название, которое видно в заголовке страницы и в `<title>`. |
| `siteTheme` | `'lumen'` | Тема оформления Bootswatch. Можно выбрать одну из 16 готовых тем (cerulean, cosmo, cyborg, darkly, flatly, journal, lumen, paper, readable, sandstone, simplex, slate, spacelab, superhero, united, yeti). |
| `disableExtraction` | `false` | Если `true` - убирает возможность конвертации в аудио (нужен ffmpeg, если его нет или не хочется тратить ресурсы на конвертацию - можно отключить). |
| `max_dl` | `1` | Сколько загрузок может идти одновременно. `-1` убирает лимит совсем (не рекомендуется - можно перегрузить сервер). |
| `outputFolder` | `/var/www/YT/download` | Абсолютный путь, куда сохраняются скачанные файлы. |
| `downloadPath` | `'download'` | Относительный веб-путь к папке загрузок - чтобы можно было скачать файл прямо по ссылке из браузера. Если оставить пустым, ссылки на скачивание не показываются (удобно, если сайт крутится локально и файлы забираются другим способом, например, по сети). |
| `redirectAfterSubmit` | `'downloads'` | На какую вкладку перекинуть пользователя после отправки ссылки на загрузку. |
| `logPath` | `/var/www/YT/tmp` | Папка, куда yt-dlp и сам сайт пишут служебные файлы: прогресс, очередь, PID-файлы. |
| `logURL` | `''` | Если эта папка доступна через веб - сюда можно прописать относительный путь, и в интерфейсе появится ссылка на лог конкретной загрузки. Пустое значение - логи не публикуются. |
| `youtubedlExe` | `/.yt_env/bin/yt-dlp` | Путь к исполняемому файлу yt-dlp. |
| `maxVideoRes` | `1080` | Потолок качества видео при обычном (не долгом) нажатии кнопки "Скачать". Долгое нажатие позволяет выбрать качество явно, не обращая внимания на этот потолок (см. [Frontend](04-frontend.md#долгое-нажатие-на-кнопку-скачать)). |
| `socks5` | из `.env` | Адрес SOCKS5-прокси. Читается через `getenv('SOCKS5_URL')`, чтобы пароль никогда не попадал в код, который лежит в git. |
| `keepPartialFiles` | `false` | Оставлять ли недокачанные `.part`-файлы после нажатия "Остановить всё". |
| `disableQueue` | `false` | Если `true` - при превышении лимита `max_dl` загрузка просто не запустится с ошибкой, вместо того чтобы встать в очередь. |
| `allowFileDelete` | `true` | Разрешить ли пользователям удалять файлы из интерфейса. |
| `showFileLifetime` | `true` | Показывать ли таймер "сколько осталось до автоудаления" у каждого файла. Само автоудаление выполняет хостовый cron (удаляет файлы старше 120 минут каждый час, см. [docs/02-setup.md](02-setup.md)), поэтому этот параметр имеет смысл только если такой cron настроен. |
| `debug` | `false` | Включает вывод всех PHP-ошибок на экран. Строго для разработки - на продакшене с доступом из интернета держать выключенным. |

## Прокси и .env

Файл `.env.example` показывает, какую переменную нужно задать:

```
SOCKS5_URL=socks5://user:pass@host:port
```

Реальный файл `.env` (не `.env.example`) не должен попадать в git - он подключается через `env_file` в `docker-compose.yml`, и Docker передаёт переменную внутрь контейнера. Внутри PHP читает её через `getenv()` прямо в `config.php`, поэтому пароль ни разу не оказывается записанным в файлах проекта.

## Прямой доступ и доступ через прокси

В классе `Downloader` (файл `class/Downloader.php`) есть список доменов `DIRECT_ACCESS_DOMAINS`. Все ссылки, чей домен совпадает с этим списком (или является его поддоменом), скачиваются напрямую, без прокси. Все остальные ссылки - например, YouTube или другие зарубежные сервисы - идут через прокси, если он настроен в `.env`.

Идея в том, что российские и не блокирующие IP сайты часто сами не очень хорошо работают через иностранные прокси (либо требуют российский IP, либо прокси у них просто медленнее прямого подключения), поэтому список явно перечисляет такие домены, чтобы не гонять их трафик через прокси без необходимости.

## Фильтр 18+ в yt-dlp.conf

Файл `app/etc/yt-dlp.conf` - общий конфиг, который yt-dlp подхватывает при каждом запуске. В нём, помимо обычных сетевых настроек (ограничение скорости, таймауты, паузы между повторами), есть один большой `--match-filter`: регулярное выражение, которое сравнивается с адресом страницы и блокирует скачивание, если домен входит в список сайтов для взрослых.

Список построен по принципу простого перечисления доменов через `|` внутри регулярки - он намеренно не идёт в эту документацию целиком (это просто длинный список адресов), важен только принцип: если ссылка ведёт на сайт из этого списка, yt-dlp откажется её скачивать ещё до старта загрузки, а интерфейс покажет пользователю сообщение "Порнографию я вам не дам".

## nginx

Конфиг лежит в `app/etc/nginx/sites-available/default`. Из значимого:
- Запросы к `.php`-файлам уходят в php-fpm через unix-сокет.
- Папки `tmp`, `class`, `config` и `.yt_env` полностью закрыты от прямого доступа через веб (`deny all`) - снаружи их вообще не видно.
- Картинки и шрифты кешируются на неделю, CSS и JS - на сутки, иконки сайтов (`favicons/`) - на год, потому что они почти никогда не меняются.
- Таймаут ответа от PHP ограничен 30 секундами - этого достаточно даже с учётом того, что на каждый запрос `index.php` дополнительно дёргает `process_queue()`.

# Установка и запуск

## Через Docker (рекомендуется)

Это самый простой способ - все зависимости (PHP, nginx, Python, Node.js, ffmpeg, сам yt-dlp) собираются внутри образа автоматически.

```bash
git clone <адрес репозитория>
cd YT
cp .env.example .env
docker-compose up -d
```

После этого сайт открывается на `http://localhost:8000` (порт задан в `docker-compose.yml`).

### Переменные окружения

В `.env` нужно указать прокси, если он нужен:

```
SOCKS5_URL=socks5://user:pass@host:port
```

Если строку оставить пустой, прокси просто не используется - все загрузки идут напрямую (кроме доменов из специального списка, см. [Backend](03-backend.md)).

Переменная читается контейнером через `env_file` в `docker-compose.yml`, а внутри PHP - функцией `getenv()` в `config.php`. Пароль от прокси никогда не попадает в код и не коммитится в git.

## Что происходит при сборке образа

`Dockerfile` устроен так:

1. Базовый образ - Debian (slim).
2. Ставятся ffmpeg, nginx, Python, PHP 8.4-fpm.
3. Отдельно подключается репозиторий NodeSource и ставится Node.js 22 - подробнее зачем, в [Архитектуре](01-architecture.md#почему-yt-dlp-требует-nodejs).
4. Через pip ставится `yt-dlp[default]` - расширенная версия с решателями JS-загадок YouTube.
5. Накатывается патч `replace_insane.patch` - он делает правила очистки имён файлов более мягкими (подробнее в [Фишках](06-features.md)).
6. nginx настраивается работать от `www-data`, прокидывает переменные окружения в php-fpm (`clear_env = no`), чтобы `SOCKS5_URL` дошла до `getenv()`.
7. Запускается `logger.sh` - он генерирует Python-плагин логирования и кладёт его в системную папку, доступную yt-dlp.
8. В конце процесс переключается на пользователя `www-data` (не root) и стартует через `start.sh`.

## Запуск без Docker

Если нужно развернуть вручную, потребуется:

- PHP 8.4 с php-fpm
- nginx (или другой веб-сервер с поддержкой PHP)
- Python 3.13
- FFmpeg
- Node.js 22+
- yt-dlp, установленный с дополнительными зависимостями (`pip install "yt-dlp[default]"`)

Дальше нужно:
1. Скопировать содержимое `app/var/www/YT` в корень веб-сервера.
2. Настроить nginx по образцу `app/etc/nginx/sites-available/default`.
3. Прописать правильные пути в `config/config.php` (`outputFolder`, `logPath`, `youtubedlExe`).
4. Запустить генератор плагина-логгера (содержимое `logger.sh`) вручную, чтобы создать Python-постпроцессор для yt-dlp.

## Скрипты

**`app/start.sh`** - точка входа контейнера. Сохраняет версию yt-dlp в файл (чтобы показать её в подвале страницы), запускает php-fpm и nginx, плюс выводит лог yt-dlp в консоль контейнера через `tail -F`.

**`app/logger.sh`** - выполняется один раз при сборке образа. Создаёт Python-файл `log_plugin.py` - постпроцессор для yt-dlp, который при каждой завершённой загрузке дописывает строку в `/var/log/yt_dlp.log`: время по Москве, имя файла, URL и IP клиента. Запись защищена файловой блокировкой (`fcntl.flock`), чтобы параллельные процессы yt-dlp не порвали лог друг другу.

**`app/etc/Scripts/cleanup.sh`** - простой скрипт, удаляющий вообще всё из `download/` и `tmp/`. Подойдёт для ручной полной очистки.

**`app/etc/Scripts/2hourcleanup.sh`** - предназначен для cron. Удаляет файлы старше 120 минут из `download/` и `tmp/` - именно так реализуется автоудаление, которое показывается пользователю как "таймер жизни файла".

**`patch_replace_insane.sh`** (в корне репозитория) - вспомогательный скрипт для тех, кто уже развернул и запустил контейнер. `Dockerfile` сам накатывает патч `replace_insane.patch` при сборке образа, но если внутри уже работающего контейнера обновить yt-dlp командой `pip install --upgrade yt-dlp`, патч слетит - новая версия yt-dlp придёт без него. Этот скрипт умеет переналожить патч прямо в работающем контейнере (`patch_replace_insane.sh container`) или в файлах соседнего репозитория на хосте (`patch_replace_insane.sh repo`), не пересобирая образ с нуля.

## Очередь без cron

Если cron не настроен, очередь всё равно продвигается: каждый раз, когда кто-то открывает страницу или браузер просто опрашивает `index.php?jobs`, вызывается `Downloader::process_queue()`. Поэтому пока хотя бы один человек держит сайт открытым во вкладке, очередь будет двигаться сама.
